# [API-001] 공통 DTO: `PersonaSetupDTO` Zod 스키마 작성

## 1. Task 개요
- **도메인:** Backend/API
- **목표:** 프론트엔드와 백엔드가 데이터를 주고받을 때 데이터가 오염되지 않도록 검증하는 Zod 스키마를 만듭니다.
- **선행 태스크:** DB-003
- **핵심 가치 (VP):** [VP-1] 무한 페르소나 설정 시 예기치 않은 오류나 버그 차단

## 2. Vibe Coding 프롬프트 (AI 입력용)

---
**[Prompt]**
이제부터 너는 TypeScript 및 API 통신 규약을 설계하는 엔지니어다.
유저가 자유롭게 **"자신만의 AI 페르소나"**를 설정할 때, 프론트엔드에서 넘어오는 데이터가 정확한지 검문하는 수문장(Zod)을 세워야 한다.

작업 지시:
1. 프로젝트 폴더 내에 `shared/types/` 또는 `lib/dto/` 디렉토리를 생성하라.
2. `persona.dto.ts` 파일을 만들고 Zod 패키지를 임포트하라.
3. `PersonaSetupDTO` 스키마를 선언하라:
   - `persona_name`: 1자 이상 50자 이하의 문자열 (필수)
   - `persona_tone`: 문자열 (필수, 유저가 입력한 말투)
   - `fcm_token`: 문자열 (선택적 속성)
4. 이 스키마에서 TypeScript 타입(`type PersonaSetupType = z.infer<typeof PersonaSetupDTO>`)을 추출하여 export 하라.
