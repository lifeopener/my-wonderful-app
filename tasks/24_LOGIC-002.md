# [LOGIC-002] 유저 인증 및 JWT Token 파싱 로직

## 1. Task 개요
- **도메인:** Backend / Edge Function
- **목표:** `chat` 함수가 호출될 때, 헤더의 JWT 인증 토큰을 검사하여 누구의 요청인지(유저 식별) 확인합니다.
- **선행 태스크:** LOGIC-001
- **핵심 가치 (VP):** 프라이버시 보장 및 보안 강화

## 2. Vibe Coding 프롬프트 (AI 입력용)

---
**[Prompt]**
아무나 우리 AI의 API를 찔러서 비용을 고갈시키면 안 된다. 유저의 요청 헤더에 있는 JWT 토큰을 뜯어서 정확히 누구인지 식별해야 한다.

작업 지시:
1. `chat/index.ts` 상단에 `@supabase/supabase-js` 라이브러리를 임포트하라.
2. 클라이언트 요청의 `Authorization: Bearer <token>` 헤더를 추출하라.
3. `supabase.auth.getUser(token)` 헬퍼 함수를 사용하여 이 토큰이 유효한지 검증하라.
4. 인증 실패 시 HTTP 401 Unauthorized 에러를 응답하라.
5. 인증 성공 시, 유저의 `id` (UUID)를 추출하여 다음 로직으로 넘겨주어라.
---

## 3. 검증 기준 (QA)
- [ ] 토큰 없이 요청을 보냈을 때 401 에러가 반환되는가?
- [ ] 올바른 토큰을 넣었을 때 유저 UUID가 정상적으로 추출되는가?
