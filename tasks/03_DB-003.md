# [DB-003] `profiles` 테이블 생성 DDL 작성

## 1. Task 개요
- **도메인:** DB Schema
- **목표:** 사용자의 닉네임, 페르소나 설정, 푸시 알림 토큰을 저장할 테이블을 만듭니다.
- **선행 태스크:** DB-001
- **핵심 가치 (VP):** [VP-1] 무한 페르소나 및 옴니채널 교감

## 2. Vibe Coding 프롬프트 (AI 입력용)

---
**[Prompt]**
우리의 핵심 가치는 **"무한한 페르소나의 설정과 옴니채널(FCM) 일상 교감"**이다. 이 철학을 담을 `profiles` 테이블을 설계해야 한다.

작업 지시:
1. `supabase/migrations/` 폴더에 새 마이그레이션 파일을 생성하라.
2. 아래 컬럼을 포함하는 `public.profiles` 테이블 DDL을 작성하라:
   - `id`: UUID (Primary Key, auth.users의 id와 외래키 연결, ON DELETE CASCADE)
   - `persona_name`: TEXT (사용자가 부르고 싶은 AI의 이름)
   - `persona_tone`: TEXT (다정하게, 차갑게 등 성격)
   - `fcm_token`: TEXT (미접속 시 선톡을 쏘기 위한 Firebase 토큰)
   - `updated_at`: TIMESTAMP (마지막 채팅 접속 시간, 선톡 발송 기준점)
   - `created_at`: TIMESTAMP
3. 파일 작성 후 로컬 DB에 마이그레이션을 적용하여 스키마가 완벽히 올라갔는지 점검하라.
---

## 3. 검증 기준 (QA)
- [ ] `profiles` 테이블이 `auth.users`와 외래키로 올바르게 묶여 있는가?
- [ ] FCM 푸시를 위한 `fcm_token` 컬럼이 존재하는가?
- [ ] 24시간 미접속 체크를 위한 `updated_at` 컬럼이 존재하는가?
