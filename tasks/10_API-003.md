# [API-003] 공통 DTO: `B2BCurationDTO` Zod 스키마 작성

## 1. Task 개요
- **도메인:** Backend/API
- **목표:** LLM이 B2B 큐레이션 정보를 반환할 때 정해진 규격(구조화된 JSON)을 무조건 지키도록 스키마를 정의합니다.
- **선행 태스크:** DB-006
- **핵심 가치 (VP):** [VP-3] 실무적 성과 창출 (유저에게 보여지는 수익화 링크의 포맷 강제)

## 2. Vibe Coding 프롬프트 (AI 입력용)

---
**[Prompt]**
AI가 유저에게 실무적 솔루션(프리미엄 레포트, 네트워킹 모임)을 제안할 때, 말로만 떠들게 두면 안 된다. 프론트엔드가 예쁜 버튼 모양으로 화면에 그릴 수 있도록, LLM의 응답을 엄격한 JSON 규격으로 강제해야 한다.

작업 지시:
1. `shared/types/` 내에 `curation.dto.ts` 파일을 생성하라.
2. `B2BCurationDTO` 스키마를 Zod로 작성하라:
   - `title`: 문자열 (제안할 솔루션 제목)
   - `category`: 문자열
   - `url`: 문자열 (반드시 올바른 URL 형식이어야 함 `z.string().url()`)
   - `description`: 문자열 (왜 이 솔루션을 추천하는지에 대한 AI의 한 줄 설명)
3. 이 스키마는 향후 Vercel AI SDK의 Tool Calling 기능에서 `parameters` 스키마로 그대로 사용될 것이므로 구조를 완벽하게 유지하라.
