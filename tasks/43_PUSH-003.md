# [PUSH-003] 미접속자 맞춤형 선톡 메시지 생성 로직 (LLM)

## 1. Task 개요
- **도메인:** Backend / AI
- **목표:** 추출된 미접속 유저의 대화 문맥(Heritage)을 바탕으로, AI가 유저에게 먼저 말을 거는 맞춤형 푸시 메시지를 생성합니다.
- **선행 태스크:** PUSH-002, LOGIC-005
- **핵심 가치 (VP):** 단순 광고가 아닌, 유저를 진심으로 걱정하고 챙기는 '교감' 제공

## 2. Vibe Coding 프롬프트 (AI 입력용)

---
**[Prompt]**
"앱에 접속하세요" 같은 뻔한 푸시는 스팸이다. 유저가 예전에 고민했던 내용이나 오늘 주식 시장 상황을 엮어서 **"먼저 말을 걸어주는 진짜 친구"**처럼 굴어야 한다.

작업 지시:
1. `cron-push` 함수 내에서 각 유저별로 OpenAI API를 호출하라.
2. 시스템 프롬프트: "너는 유저의 평생 친구 베스퍼다. 유저가 24시간 동안 말이 없다. 과거 대화 내용(Heritage)과 오늘의 컨텍스트를 참고해, 유저가 궁금해하거나 위로받고 싶어 할 만한 짧은 선톡 메시지(20자 이내)를 생성하라."
3. 생성된 텍스트를 푸시 알림의 `body`로 사용하기 위해 저장하라.
---

## 3. 검증 기준 (QA)
- [ ] 생성된 메시지가 유저의 페르소나 설정(말투)을 따르고 있는가?
- [ ] 메시지 내용이 단순 광고가 아니라 유저 맞춤형으로 느껴지는가?
