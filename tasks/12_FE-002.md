# [FE-002] React Navigation (라우팅) 셋업

## 1. Task 개요
- **도메인:** Frontend / UI
- **목표:** 앱 내 화면 간의 부드러운 전환을 담당하는 네비게이션 구조(Stack & Bottom Tab)를 셋업합니다.
- **선행 태스크:** FE-001
- **핵심 가치 (VP):** NFR-000 제약(억압적 UI 배제) 준수 및 부드러운 화면 전환으로 사용자 몰입감 유지.

## 2. Vibe Coding 프롬프트 (AI 입력용)

---
**[Prompt]**
모바일 앱의 생명은 '길 찾기(Navigation)'다. 화면이 뚝뚝 끊기거나 갇혀버리는 느낌(억압적 UI)을 주면 절대 안 된다.

작업 지시:
1. `@react-navigation/native` 및 `@react-navigation/native-stack` 패키지를 설치하라. (Expo 전용 의존성도 함께 설치)
2. 앱의 루트를 래핑하는 `NavigationContainer`를 설정하라.
3. 기본 Stack Navigator를 생성하고, 임시로 두 개의 더미 스크린(`HomeScreen`, `ChatScreen`)을 연결하라.
4. 스크린 전환 애니메이션이 기본 네이티브 전환(iOS 슬라이드, Android 페이드)을 따르도록 설정하라.
5. 버튼을 눌러 `HomeScreen`에서 `ChatScreen`으로 정상 이동하는지 확인하고 코드를 제출하라.
---

## 3. 검증 기준 (QA)
- [ ] NavigationContainer가 App.tsx 또는 루트 컴포넌트를 감싸고 있는가?
- [ ] 스택 이동(`navigation.navigate`) 시 에러가 발생하지 않는가?
- [ ] 뒤로 가기 제스처(iOS 엣지 스와이프 등)가 정상 작동하는가?
