# [DB-007] Row Level Security (RLS) 방어벽 구축

## 1. Task 개요
- **도메인:** DB Security
- **목표:** 사용자가 오직 '자신의 프로필'과 '자신의 대화 기록'에만 접근할 수 있도록 DB 자체 방어벽(RLS)을 칩니다.
- **선행 태스크:** DB-003, DB-004
- **핵심 가치 (VP):** 철저한 프라이버시 보장을 통한 절대적 심리적 락인(Lock-in) 완성

## 2. Vibe Coding 프롬프트 (AI 입력용)

---
**[Prompt]**
우리의 앱은 유저의 가장 내밀한 두려움과 투자 성향을 담는 공간이다. **"절대적 프라이버시"**가 생명이다. 서버 로직의 실수가 있더라도 DB 단에서 남의 데이터를 볼 수 없게 원천 차단해야 한다.

작업 지시:
1. 새 마이그레이션 파일을 생성하라.
2. `profiles`, `heritage_logs`, `api_usage_logs` 테이블에 대해 Row Level Security를 활성화하라 (`ALTER TABLE ... ENABLE ROW LEVEL SECURITY;`).
3. 사용자가 `auth.uid()` 와 테이블의 `id` (또는 `user_id`)가 일치할 때만 SELECT, INSERT, UPDATE, DELETE가 가능하도록 Policy를 작성하라.
4. (중요) Edge Function(서버)에서 서비스 롤(Service Role) 키로 접근할 때는 RLS를 우회할 수 있도록 설정이 맞는지 점검하라.
5. 로컬 DB에 스키마를 적용하라.
