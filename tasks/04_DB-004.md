# [DB-004] `heritage_logs` 테이블 생성 DDL 작성

## 1. Task 개요
- **도메인:** DB Schema
- **목표:** 유저와 AI 양방향의 모든 대화 기록을 1536차원 벡터와 함께 영구 저장할 테이블을 만듭니다.
- **선행 태스크:** DB-002
- **핵심 가치 (VP):** [VP-2] 나이를 먹듯 유저의 패턴을 학습하여 진화하는 지능 (RAG)

## 2. Vibe Coding 프롬프트 (AI 입력용)

---
**[Prompt]**
우리의 AI는 시간이 지날수록 유저의 환희와 공포를 학습하는 **"초밀착 인지 동행 파트너"**가 되어야 한다. 단순 텍스트 기록이 아닌, 모든 대화의 '의미(벡터)'를 저장할 공간이 필요하다.

작업 지시:
1. 새 마이그레이션 파일을 만들고 `public.heritage_logs` 테이블 DDL을 작성하라.
2. 다음 컬럼을 포함해야 한다:
   - `id`: UUID (Primary Key)
   - `user_id`: UUID (외래키 -> profiles.id)
   - `role`: TEXT ('user' 또는 'assistant' 만 허용하는 CHECK 제약조건 필수)
   - `content`: TEXT (대화 원문 텍스트)
   - `embedding`: VECTOR(1536) (OpenAI 임베딩 사이즈에 맞춘 벡터 타입)
   - `created_at`: TIMESTAMP
3. **주의사항:** 유저의 말뿐만 아니라 'Assistant'의 말도 반드시 기록되어야 진정한 양방향 진화가 가능하다. `role` 컬럼 제약조건을 꼼꼼히 확인하라.
4. 로컬 DB에 마이그레이션을 적용하라.
---
