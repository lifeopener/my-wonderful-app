# [DB-012] `seed.sql` UI 테스트용 더미 데이터 주입

## 1. Task 개요
- **도메인:** Mocking
- **목표:** 개발 초기 단계에서 UI가 잘 작동하는지 확인하기 위해 더미 유저, 대화 기록(Heritage), B2B 큐레이션 데이터를 DB에 미리 채워넣습니다.
- **선행 태스크:** DB-010, DB-006
- **핵심 가치 (VP):** 빠른 개발 반복 주기 확보

## 2. Vibe Coding 프롬프트 (AI 입력용)

---
**[Prompt]**
텅 빈 앱은 개발하기 힘들다. 풍부한 대화 기록과 카탈로그가 들어있는 테스트 환경을 만들어달라.

작업 지시:
1. `supabase/seed.sql` 파일을 생성하거나 수정하라.
2. 테스트용 유저 프로필 1개와 그 유저가 과거에 나눈 대화 기록(`heritage_logs`) 10건 이상을 INSERT 하라. (임베딩 값은 임의의 벡터로 채워라)
3. 교육, 네트워킹, 투자 카테고리별 B2B 큐레이션 데이터를 각 3건씩 총 9건 INSERT 하라.
4. `supabase db reset` 명령어로 데이터가 정상적으로 초기화 및 로드되는지 확인하라.
---

## 3. 검증 기준 (QA)
- [ ] `supabase db reset` 후 DB를 열었을 때 더미 데이터들이 보이는가?
- [ ] `heritage_logs`의 텍스트 내용이 투자/성장 테마에 맞게 자연스러운가?
