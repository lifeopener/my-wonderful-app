# [DB-005] `api_usage_logs` 추적 테이블 및 View 생성

## 1. Task 개요
- **도메인:** DB Schema
- **목표:** 유저가 AI API를 호출할 때마다 소모되는 비용(토큰)을 추적하고, 월간 총액을 계산합니다.
- **선행 태스크:** DB-001
- **핵심 가치 (VP):** 예산 통제 및 [VP-3] 안정적인 실무 서비스 유지 (5만원 한도 방어)

## 2. Vibe Coding 프롬프트 (AI 입력용)

---
**[Prompt]**
우리의 시스템은 훌륭하지만, 무제한 API 호출로 인해 월 예산(5만 원)이 뚫리면 프로젝트는 파산한다. 이를 방어하기 위한 비용 감시 테이블이 필요하다.

작업 지시:
1. 새 마이그레이션 파일을 생성하고 `public.api_usage_logs` 테이블 DDL을 작성하라.
2. 다음 컬럼을 포함하라:
   - `id`: UUID
   - `user_id`: UUID
   - `endpoint`: TEXT ('chat', 'embedding', 'search' 만 허용)
   - `token_count`: INT (사용 토큰 수)
   - `estimated_cost_krw`: NUMERIC(10,4) (원화 환산 추정 비용)
   - `created_at`: TIMESTAMP
3. 이 테이블을 기반으로, 현재 '월(Month)'의 총 사용 금액을 실시간으로 계산해 보여주는 `public.monthly_cost_summary` VIEW를 작성하라.
4. 로컬 DB에 스키마를 적용하고 에러가 없는지 확인하라.
---
