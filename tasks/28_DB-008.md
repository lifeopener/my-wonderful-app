# [DB-008] `heritage_logs` 테이블 Row Level Security 정책 적용

## 1. Task 개요
- **도메인:** Security
- **목표:** `heritage_logs` 테이블에 RLS를 적용하여 사용자가 자신의 대화 기록만 조회하고 생성할 수 있도록 보안을 강화합니다.
- **선행 태스크:** DB-004
- **핵심 가치 (VP):** [VP-4] 초밀착 인지 동행 (개인 정보의 절대적 보호)

## 2. Vibe Coding 프롬프트 (AI 입력용)

---
**[Prompt]**
유저의 대화 기록은 가장 민감한 자산이다. DB 단에서 완벽하게 격리되어야 한다.

작업 지시:
1. `supabase/migrations/`에 `heritage_logs` RLS 정책을 설정하는 마이그레이션 파일을 생성하라.
2. `heritage_logs` 테이블에 RLS를 활성화하라.
3. `SELECT`, `INSERT` 시 `auth.uid() = user_id` 인 경우만 허용하는 Policy를 작성하라.
4. `UPDATE`, `DELETE`는 원칙적으로 금지하거나 동일한 소유자 체크를 적용하라. (우리의 철학상 대화 기록은 수정/삭제가 기본적으로 필요 없음)
---

- [ ] `auth.uid()`가 없는 익명 요청이 차단되는가?
