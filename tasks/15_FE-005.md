# [FE-005] 공통 컴포넌트: 에러/로딩 처리 Fallback UI

## 1. Task 개요
- **도메인:** Frontend / UI
- **목표:** 네트워크 지연이나 에러 발생 시 앱이 빈 화면으로 죽지 않고 우아하게 복구(Graceful Degradation)하는 UI를 만듭니다.
- **선행 태스크:** FE-001
- **핵심 가치 (VP):** 유저에게 불안감을 주지 않는 안정적인 초밀착 동행 환경 구성 (NFR-000 준수)

## 2. Vibe Coding 프롬프트 (AI 입력용)

---
**[Prompt]**
앱이 먹통이 되면 유저는 두려움을 느낀다. 우리의 철학(NFR-000)은 사용자의 심리를 통제하거나 불안하게 만들지 않는 것이다. 데이터 로딩이나 에러 발생 시 부드러운 대처 UI가 필요하다.

작업 지시:
1. `components/LoadingSpinner.tsx`를 생성하여, NativeWind 애니메이션(`animate-spin`)을 활용한 부드러운 로딩 인디케이터를 만들어라.
2. `components/ErrorBoundaryUI.tsx`를 생성하여, 치명적 에러가 발생했을 때 앱을 크래시 내지 않고 "네트워크가 잠시 불안정합니다"라는 친절한 문구와 '다시 시도' 버튼을 보여주는 뷰를 만들어라.
3. 화면 전체를 막는(Blocking) 디자인은 피하고, 컨텐츠 영역 내부에서만 스피너가 돌도록 스타일링하라.
---

## 3. 검증 기준 (QA)
- [ ] 로딩 인디케이터가 버벅임 없이 60fps로 부드럽게 돌아가는가?
- [ ] 에러 화면에 '다시 시도(Retry)' 버튼이 명확히 존재하며 클릭 가능한가?
- [ ] 화면 전체를 무조건 블러 처리하거나 터치를 억압하는 구조가 아님이 확인되었는가?
